## 赛道

### 赛道一：Cobo 赛道｜Agentic Economy × Cobo Agentic Wallet

**关注方向：** Agentic Commerce — 构建让 AI 参与经济活动的应用或基础设施。

**建议方向：**
1. **Agent-Native Payments** — 让 Agent 成为互联网的一等支付公民：HTTP 402 自动完成支付，不依赖 API Key 和人类预注册
2. **Trustless Agent Work Agreements** — 基于 ERC-8183 实现 Agent 间去信任工作协议：发布→托管→交付→验收/驳回→付款
3. **Agent Resource Procurement** — Agent 根据任务自主发现、比价、采购算力、数据、API、存储等资源
4. **Autonomous Trading Agent** — 自主执行链上交易策略：套利、做市、组合管理、流动性挖矿
5. **A2A Economy** — 多个 Agent 各自拥有钱包→Agent-to-Agent 经济体：互雇、拍卖、分账、管理 Treasury

**参考资料：**
- 官网：https://www.cobo.com/agentic-wallet
- Recipes：https://agenticwallet.cobo.com/agentic-wallet/recipes
- 文档：https://www.cobo.com/products/agentic-wallet/manual/start-here/introduction
- SDK：https://www.cobo.com/products/agentic-wallet/manual/developer/quickstart-overview

**赛道规则：**
- 项目必须围绕 Agent 与资金操作场景展开
- Agent 的资金相关操作需通过 Cobo Agentic Wallet（CAW）完成
- Agent 需要具备真实的资金执行能力
- 项目需体现 CAW 在 Agent 钱包管理、权限控制、安全隔离或自主支付等方面的价值
- 最终成果需为可运行或可演示的产品原型（Demo），不接受纯 PPT、概念设计或 Mockup

**提交要求：**
- GitHub Repo
- README 与项目说明文档
- Demo 视频（建议 3–5 分钟）
- 项目演示链接（如有）
- 使用 CAW 的关键代码或配置说明
- 如涉及链上交互，建议提供：测试网地址、Transaction Hash、Agent Wallet 地址、流程截图或操作记录

**Cobo 赛道评审侧重点：**
- 场景贴合度：项目是否清楚体现 Agentic Commerce
- CAW 关键性：CAW 是否是项目资金流程中的关键组件
- 资金流程完整度：Demo 是否清楚展示 Agent 从任务触发到资金操作完成的主要流程
- 可演示性：Demo 是否能稳定展示核心流程
- 风险边界说明（如适用）

### 赛道二：Z.AI 赛道｜Web3 × Long-Horizon Task

**关注方向：** GLM-5.1 的长程任务能力 — 让 Agent 自主拆解复杂任务、制定多步骤计划、持续调用工具、迭代修复，并完成从需求到交付的完整 Web3 工作流。

**建议方向：**
1. **Agentic Dev Tools for Web3** — 自然语言驱动完整 Web3 工程：需求分析→合约设计→代码编写→测试迭代→部署
2. **AI-Powered 3D World Builder** — 从空间规划、场景生成、资产配置到可部署包的完整生产链路
3. **AI × Creator Economy** — 从创意输入到链上变现的完整内容生产链路

**参考资料：**
- Z.AI 开发者文档：https://docs.z.ai/devpack/overview
- API 接入：https://docs.z.ai/api-reference/introduction
- GLM-5.1 技术报告：https://z.ai/blog/glm-5.1
- 开源模型：https://huggingface.co/zai-org/GLM-5.1
- Coding Plan 折扣：https://z.ai/subscribe?ic=8JVLJQFSKB（10% off）

**赛道规则：**
- 推荐使用 GLM-5.1 完成；Demo 中 Agent 的核心任务应由 GLM-5.1 驱动
- 项目需围绕 Web3 × Long-Horizon Task 展开，体现长程、自主、持续执行能力
- Agent 需要能自主拆解复杂任务、制定并执行多步骤计划，并持续迭代或自我纠错
- 最终成果需为可运行或可演示的产品原型（Demo）

**提交要求：**
- GitHub Repo 与 README：说明项目目标、架构、运行方式、GLM-5.1 调用位置与关键流程
- 可运行 Demo 或可访问演示链接；建议提供 3–5 分钟 Demo 视频
- 长程任务运行记录：展示 Agent 的任务拆解、工具调用、迭代/修复过程和最终交付成果
- Web3 相关证明：如合约地址、测试网交易、NFT/链上资产等

**Z.AI 赛道评审侧重点：**
- 赛道匹配度：项目是否围绕 Web3 × Long-Horizon Task 展开
- GLM-5.1 使用关键性：Agent 的核心长程任务是否由 GLM-5.1 驱动
- 任务复杂度与闭环程度：是否覆盖从需求理解到交付的完整链路
- 长程稳定性：多步骤执行中 Agent 是否能保持目标一致
- Web3 价值：是否解决 Web3 开发、3D 设计、内容生产等真实问题
- 可演示性与可复现性
